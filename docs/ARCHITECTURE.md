## Key Layers

### 1. Container Parser (`src/container/`)

Reads and writes the 7z archive structure:
- **signature_header.rs** — 32-byte magic + version + CRC checks + next-header offset
- **header.rs** — property stream (folders, streams, file names, timestamps, sizes)
- **folders.rs** — folder structure + unpacking sizes + bond resolution
- **coders.rs** — coder method ID registry + method-specific properties
- **properties.rs** — variable-length integers (UINT64) and bit vectors
- **crc.rs** — CRC-32 validation

This layer knows **nothing** about actual compression — only the 7z metadata structure.

### 2. Pipeline / Dispatch (`src/pipeline/`)

Routes decoded bytes to the right codec implementation:
- **dispatch.rs** — method-ID → trait-object Coder (the **ONE place** each sub-crate plugs in)
- **coder_trait.rs** — `trait Coder { fn decode(...) -> Result<...>; fn encode(...); }`
- **copy.rs** — in-tree Copy coder (identity operation; proof of concept)

### 3. Codec Implementations (Sub-crates)

Each algorithm lives in its own repo + crate, implementing the `Coder` trait:

```rust
impl Coder for LzmaCoder {
    fn decode(&self, input: &[u8], properties: &[u8]) -> Result<Vec<u8>> {
        // Decompress LZMA bytes
    }
    fn encode(&self, input: &[u8], properties: &[u8]) -> Result<Vec<u8>> {
        // Compress to LZMA
    }
}
```

Sub-crate repos are public and independent (so they can be published to crates.io), but 7zippy depends on them as ordinary `[dependencies]` entries (feature-gated).

## Data Flow: Decompress

```
user calls
  sevenzippy::read_archive("file.7z") → read Container → iterate Folders
    for each Folder:
      parse Coder chain from Container
      Pipeline::dispatch(MethodId) → Coder trait object
      call Coder::decode(packed_bytes, properties) → unpacked bytes
    collect all unpacked bytes into files
  return Archive { files: [...] }
```

## Data Flow: Compress

```
user calls
  sevenzippy::write_archive(files, CoderSpec) → create Container
    for each file:
      Pipeline::dispatch(CoderSpec::method_id) → Coder trait object
      call Coder::encode(bytes, properties) → packed bytes
      add to Folder with metadata (sizes, hashes)
    finalize Folder + Header
    serialize Container to disk
```

## Test Layers (`src/tests/`)

The test suite is organized by **data complexity** — not by module:

1. **fixtures.rs** — deterministic data generators (zeros, random, sequential)
2. **layer0_generators.rs** — validate the fixture generators themselves
3. **layer1_container.rs** — parse a known-good 7z archive (generated by 7zz)
4. **layer2_dispatch.rs** — route the correct method ID → Coder
5. **layer3_per_coder.rs** — smoke tests per codec (call Coder trait, no real impl yet)
6. **layer4_pipeline.rs** — full folder pipeline (Copy-only; lights up as codecs land)
7. **layer5_cross.rs** — 7zippy ↔ 7zz round-trip compatibility
8. **layer6_threads.rs** — placeholder; threading comes later
9. **layer7_perf.rs** — throughput bounds (informational, off by default)
10. **oracle.rs** — 7zz harness (compress/decompress via CLI)

Each test references `corpora/fixtures/` files committed to the repo (< 1 MB total). Oracle tests use temporary directories and shell out to `7zz`.
