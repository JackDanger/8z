# CLAUDE.md — 8z Development Guide

## Prime Directive

**8z aims to be a complete, correct, and fast pure-Rust 7z archive implementation.**

## Absolute Rules

1. **ONE PRODUCTION PATH** — know exactly which function the CLI calls. Test that function.
2. **RUN `make` BEFORE PUSH** — `make` catches regressions in < 1 minute.
3. **UPDATE `STATUS.md` IN EVERY PR** — it is the source of truth for our position. Every commit touching `src/` must update the corresponding row in `STATUS.md`.
4. **ORACLE TESTS ARE NOT OPTIONAL** — all codec implementations must pass round-trip tests against the `7zz` CLI. Missing oracle tests = incomplete work.
5. **OPUS DELEGATES, SONNET AND HAIKU IMPLEMENT** — this repo uses an agent orchestration model. Opus (oversight-only) reads `STATUS.md`, dispatches agents, and reviews. Sonnet (judgment work) and Haiku (mechanical work) do all implementation. See the [Iteration Loop](#iteration-loop) below and `memory/8z-opus-oversight-only.md`.

## Iteration Loop

1. **Read STATUS.md** — understand current focus and next ⬜ row.
2. **Pick the next incomplete slice** — LZMA range coder? LZMA literal decoder? Container header fixes?
3. **Opus dispatches an agent** — Haiku for mechanical work (templates, renames), Sonnet for judgment (parsing, trait design, test harness).
4. **Agent reports back** — summary of files touched, verification (`cargo test`, `make oracle-check`).
5. **Opus reviews and updates STATUS.md** — promote ⬜ to 🟡 or ✅.
6. **Repeat** — next ⬜ becomes the focus.

See [docs/ITERATION_LOOP.md](docs/ITERATION_LOOP.md) for the full recipe.

## Hard-Won Lessons

**When in doubt, ask the 7zz CLI what it produces — it's the canonical reference.**

We validate against Igor Pavlov's 7-Zip (`7zz`) in every round-trip test. If our output doesn't match 7zz's, we're wrong. If 7zz rejects our archive, it's broken. The oracle harness (`src/tests/oracle.rs`) makes this cheap: write a fixture, call `oracle::seven_zip_compress(…)`, read back the bytes, assert equality.

## Branch and PR Workflow

**main is protected** — no direct pushes. Every change goes through a PR (enforced by pre-push hook).

```bash
# Start work
git checkout -b feat/my-feature   # or fix/, refactor/, etc.

# Iterate
cargo test --workspace --release
make

# Before push: update STATUS.md if touching src/
# Commit and push
git push origin feat/my-feature
gh pr create --fill               # title + body from commits
# CI runs → merge when green
```

Emergency bypass (never for correctness issues):
```bash
git push --no-verify origin feat/my-feature   # skip local hook only
```

## Verification Checklist

Before finishing a PR:
- [ ] `cargo fmt --check`
- [ ] `cargo clippy --workspace --all-targets -- -D warnings`
- [ ] `cargo test --workspace --release`
- [ ] `make oracle-check` (if implementing a new coder)
- [ ] `STATUS.md` updated (if touching `src/`)
- [ ] `make status` shows correct current focus

If you're implementing a new coder:
- [ ] Fixture `.7z` file committed to `corpora/fixtures/archives/` (generated by `7zz`)
- [ ] Oracle round-trip test passing (compress with `7zz`, decompress with our code, assert equal; then vice versa)
- [ ] Integration test added to `src/tests/layer3_per_coder.rs` or `layer4_pipeline.rs`
- [ ] STATUS.md row promoted from ⬜ to ✅
